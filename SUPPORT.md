# Support

## Supported versions

- **Latest release**: full support.
- **N-1 minor release**: security and critical fixes only.
- **Older releases**: no active support.

See `docs/SUPPORT_POLICY.md` for detailed lifecycle and EOL policy.

## Get help

- **Questions / usage help**: GitHub Discussions.
- **Defects / regressions**: GitHub Issues.
- **Security vulnerabilities**: private report flow in `SECURITY.md`.

## What to include in support requests

- AegisBPF version and deployment mode (`audit` / `enforce`).
- Kernel version, distro, architecture.
- Minimal policy sample and exact command used.
- Relevant logs and metrics snippets (sanitized).
